🌐 GitHub Pages 設定メモ

GitHub Pages を使って静的サイト（今回の学習ブログ）を公開した流れとつまづきポイントのまとめ。

1. GitHub リポジトリ作成

  • リポジトリ名：YoshiLog
  • リポジトリの公開設定をパブリックに設定。

2. GitHub Pages の設定

  • GitHub のリポジトリページへ。
  • Settings → Pages を開く。
  • ブランチを master、フォルダは /(root)に設定。
  • 保存すると、以下のような URL が表示される。

    • https://yoshinpey.github.io/YoshiLog/

    アクセスするとトップページに飛ぶ。

3. ファイル構成の注意点

  • 初期ファイルの名前は必ずindex.htmlにしましょう！

    • ※indexファイルがないと「404 Not Found」エラーになります。

  • トップページを変更したい場合、index.html内で別ファイルを読み込むことで間接的にアクセス可能です。

4. Git コマンド操作

基本的な流れ：

git init
git remote add origin https://github.com/yoshinpey/YoshiLog.git
git pull origin master
git add .
git commit -m "初回コミット"
git push origin master

5. よくあるトラブル

  • index.html が無い： GitHub Pages はトップに index.html を探すので用意
  • ブランチが None： GitHub Pages の設定で master を選ぶ必要あり
  • 反映されない： 少し時間がかかることもある（数分待つ）

6. テストページ確認

  • test.html

    • ※↑実際のファイル名はちゃんとindex.htmlにしてね。
